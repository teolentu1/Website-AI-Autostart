No-GUI Game Graph Representation

Structure of the Game Graph

While interacting with the graph of the game, the game is represented as a standard graph. Each node is represented as a Node class. A reference to the whole graph is simply a reference to the root node. The adjacency_list takes EnumLR as its key — an enum representing left and right options — so each node has at most 2 references to other nodes.

class Node:
   def __init__(self, text: str, left_option: str = "", right_option: str = ""):
       self.id: int = id(self)
       self._text = text
       self.left_option = left_option
       self.right_option = right_option
       self.adjacency_list: dict[EnumLR, Node] = {}
       self.is_win: bool = False
       self.is_losing: bool = False

This is a natural representation of a graph, but in some instances a graph like this becomes a recursive data structure (when node A points to node B and node B points to node A). This imposes a challenge of serialising the data structure into a standardised format like JSON.

We have therefore implemented a serialised version of a node called SerialNode:

class SerialNode(BaseModel):
   id: int
   text: str
   left_option: str = ""
   right_option: str = ""
   adjacency_list: dict[EnumLR, int]
   is_win: bool = False
   is_losing: bool = False

SerialNode holds the same data as Node, but does not include direct references to other serial nodes — instead it indirectly references other nodes by their corresponding ids.

All nodes are then put into a dictionary where each key is the serial node's id and the value is the actual SerialNode. This data structure is called a SerialGraph:

class SerialGraph(BaseModel):
   nodes: dict[int, SerialNode]

Both classes implement the BaseModel class from the Pydantic library, inheriting the utility methods model_validate_json (convert JSON to the class), model_dump_json (convert the class to JSON), and model_json_schema (used in LLM structured output generation).

Converting Between Serial and Standard Graph

Conversion between the serialised and standard graph assumes:

The game graph is a fully connected graph, and every node is eventually reachable from the root node.

This is an acceptable assumption since gameplay always starts from the root node, and any node that is meaningful in the game must be reachable from it.

Serialisation of Graph

Serialising an individual node is straightforward — SerialNode holds the same data as Node but stores adjacent node ids rather than direct references. This method is used internally by SerialGraph to serialise the whole graph:

@classmethod
def serialize_node(cls, node: Node) -> Self:
   return SerialNode(
       id=node.get_id(),
       text=node.getText(),
       left_option=node.left_option,
       right_option=node.right_option,
       adjacency_list={side: adjacent_node.get_id() for
                       side, adjacent_node in
                       node.adjacencyList.items()},
       is_win=node.is_win,
       is_losing=node.is_losing
   )

SerialGraph includes a class method serialize_graph. Starting from the root node, it uses a depth-first search to traverse the whole game graph, storing each converted node as a SerialNode in the SerialGraph's node dictionary:

@classmethod
def serialize_graph(cls, root: Node) -> Self:
   serial_graph: SerialGraph = SerialGraph(nodes={})

   def dfs(node: Node):
       if node.get_id() in serial_graph.nodes:
           return
       serial_graph.nodes[node.get_id()] = SerialNode.serialize_node(node)
       for adjacent_node in node.adjacencyList.values():
           dfs(adjacent_node)

   dfs(root)
   return serial_graph

The serialised graph is then ready to be saved to a JSON file using Pydantic's model_dump_json method.

Deserialisation of Serial Graph

Deserialising a serial graph from JSON into a playable standard graph is done in two iterations over the SerialGraph's nodes dictionary.

In the first iteration, all individual nodes are converted from SerialNode to Node and stored in an id-to-node dictionary alongside the root node. In the second iteration, connections between these nodes are established, producing the final deserialised graph referenced by the root node:

@classmethod
def deserialize_graph(cls, serial_graph: Self) -> Node:
   root, nodes = SerialGraph._load_nodes(serial_graph)
   SerialGraph._establish_connections(serial_graph, nodes)

   return root
