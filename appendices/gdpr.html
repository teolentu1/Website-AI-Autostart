GDPR and Legal Compliance

This section outlines the legal considerations for the MotionInput Games project, including GDPR compliance, intellectual property, software licensing, and data governance. The project consists of AI-AutoStart, a gesture-based desktop automation system, and No GUI Game Engine, a local-first interactive game using gesture and keyboard input. Both systems are designed as proof-of-concept solutions that may later be deployed in schools, companies, or public environments.

The project follows legal and ethical guidelines related to data protection, software ownership, and responsible system deployment. A separate Privacy Policy document provides detailed information about data collection, storage, and user rights, while this section focuses on governance, legal responsibilities, and compliance.

GDPR and Data Privacy

The MotionInput Games project follows GDPR principles of data protection by design and by default. The system architecture is designed to minimize legal and privacy risks through local-first processing and transparent user control over system functionality.

Rather than storing or transmitting user data externally, the project ensures that processing is limited to what is necessary for system operation. Users remain in control of their data and system usage, and transparency is maintained through the dedicated Privacy Policy, which explains how data is processed and managed.

The project aligns with key GDPR principles:

  • lawfulness, fairness, and transparency
  • data minimization
  • user consent and control
  • secure local processing
  • accountability and governance

This approach ensures that the system remains compliant with GDPR requirements and suitable for future deployment in regulated environments such as schools or companies.

Data Governance

The MotionInput Games project team acts as the data controller and is responsible for ensuring that the system complies with legal and ethical standards. This includes maintaining privacy documentation, ensuring transparency, and managing any future deployment requirements.

The lawful basis for processing is user consent, as users voluntarily choose to enable system features and interact with the application. Since the system operates locally and does not involve third-party processors or cloud services, governance responsibilities remain within the project team.

This structure ensures clear accountability and supports GDPR compliance through responsible system management and documentation.

Source Code License

The project is released under the MIT License (Copyright 2026 – MotionInput Games).

This license allows users to freely use, modify, and distribute the software, provided that the copyright notice and license are included in all copies. The software is provided “as is,” without warranty, and the developers are not liable for damages resulting from its use.

Using the MIT License promotes openness and collaboration while ensuring that intellectual property ownership remains with MotionInput Games.

Intellectual Property

The intellectual property of the project includes:

  • AI-AutoStart system
  • No GUI Game Engine
  • source code and architecture
  • gesture recognition logic
  • documentation and design

These components are owned by MotionInput Games, while third-party libraries such as OpenCV and MediaPipe remain under their respective licenses.

All external components are properly licensed and used in accordance with open-source regulations, ensuring compliance with intellectual property law and software licensing requirements.

Accessibility and Legal Compliance

The project promotes responsible and inclusive software development by supporting gesture-based interaction and alternative input methods, making the system suitable for different user environments such as schools and public institutions.

Legal risks are minimized through:

  • clear privacy policy and documentation
  • defined data governance structure
  • open-source licensing
  • responsible use of third-party libraries
  • ethical and transparent system design

This ensures that the system can be deployed in a legally compliant and responsible manner.
