Critical Evaluation of the Project

UI/UX

The user interface and user experience of both AI-Autostart and Soundlab Games is minimalistic and intuitive.

In SoundLab Games, the Game Editor provides a clean canvas for node-based creation, while the Game Player presents only essential controls required to load and run a game. In AI-Autostart, all processes and pages are clearly labeled for ease-of-use without extra clutter.

Long-running operations such as AI generation and saving are handled in the background to maintain responsiveness. The interface provides clear visual feedback during these processes, improving usability despite computationally intensive tasks.

Overall, we believe the UI/UX design is excellent and highly suitable for our goal and audiences.

Functionality

Based on our summary of achievements, all compulsory features for both projects have been implemented. There are a few non-critical functionalities remaining and a couple minor bugs, however based on our unit tests and extensive user testing, our products are fully usable.

Stability

The app performed well under unit testing and user testing.

AI-Autostart demonstrated stable and accurate gesture recognition and consistent application launching during extended usage sessions. SoundLab Game Editor also maintained stable performance during game creation and saving.

Overall, both applications maintained consistent performance without unexpected behaviours, demonstrating full usability in real-time environments.

Efficiency

Both systems were designed with efficiency in mind in order to run smoothly on Intel hardware.

AI-Autostart performs gesture recognition consistently in an efficient manner that doesn't use up a lot of resources. The system tray app is lightweight and background processing ensures that recognition and system updates do not interfere with user interaction.

SoundLab Games efficiently generates and loads audio files and game structures, allowing smooth gameplay without any delays. AI generation and audio synthesis are computationally intensive by nature. Despite that our performance testing shows that CPU usage remains high during local LLM inference and the UI remains responsive due to background threading. Therefore we can say that the system performs within expected bounds.

Overall, the systems demonstrate good computational efficiency for their intended use.

Compatibility

Our application runs on any Windows 11 device on Intel Hardware. Both AI-Autostart and SoundLab Games are compiled into executable applications, allowing them to run independently without requiring additional software installations or complex configuration.

AI-Autostart is also compatible with a range of older hardware with less resources due to its lightweight nature. This makes it perfect for integrating the system for self-service kiosk use.

SoundLab Games' local LLM does not require dedicated GPUs and can operate using CPU-based inference.

Our systems are easy to deploy, install, and use on standard Windows machines, particularly in real-life environments.

Maintainability

Throughout the project development, we aimed to produce clean and well-structured code. Each class is responsible for one distinct action, following encapsulation principles, making our codebase readable and providing easier debugging and testing.

The separation of AI-Autostart and SoundLab Games into independent systems also enhances maintainability and reduces system complexity. Additionally, clear folder structures, consistent naming conventions, and version control through GitHub ensure that future developers can easily understand and extend the system.

To summarise, the system is well-structured and maintainable in the long term, allowing for future improvements and scalability.

Project Management

We utilised a variety of tools to effectively manage this project. Our team communicated through WhatsApp and GitHub, and held regular in-person meetings in addition to the lab hours. For research, presentations and shared resources we used a shared Google Drive in order to keep everything organized. All code collaboration was handled through two dedicated GitHub repositories with issues, pull requests and commit history.

Ultimately, the team demonstrated a well-organised and efficient workflow throughout the project, reflecting strong project management practices.

One team member was inactive for a significant portion of the project, and therefore the evaluation of teamwork and project management reflects the contributions and coordination of the actively participating members.
