TTS Implementation

A text-to-speech model has been used to generate audio for the SoundLab Games. The model used in this project is hexgrad/Kokoro-82M [1].

The Hexgrad team provides a kokoro package that allows for easy integration of the model into python programs. The package offers multiple language codes, as well as multiple speaker voices in the given language code.

Talker is the public facing class responsible for generation of text-to-speech audio. The Kokoro pipeline is lazily loaded, since we have found that it has been a major performance bottleneck for the startup of the SoundLab Game engine.

class Talker:
   def __init__(self):
       # 'b' - for British English
       self.pipeline: KPipeline | None = None


   def _get_pipeline(self) -> KPipeline:
       if self.pipeline is None:
           self.pipeline = KPipeline(lang_code='b', repo_id="hexgrad/Kokoro-82M")
       return self.pipeline

As can be seen in the _get_pipeline method, the language code is one of the pipeline constructor arguments. Currently, the language code is set to 'b', standing for British English. This decision has been made after a discussion with the client. Kokoro offers 8 British English speakers, 4 male and 4 female. Their corresponding codes can be found here: VOICES.md. The codes for the British English speakers are used by frontend to request a specific voice to transcribe the text.

The audio generation itself is performed in the public facing method generate_speech. This method takes in 3 arguments:

  • text - text to transcribe
  • output_file
  • voice - the voice of the speaker transcribing
def generate_speech(self, text: str, output_file: str, voice: str):
   generator = self._get_pipeline()(text, voice=voice)


   audio_chunks = []
   for gs, ps, audio in generator:
       audio_chunks.append(audio)


   if audio_chunks:
       import numpy as np
       full_audio = np.concatenate(audio_chunks)
       sf.write(output_file, full_audio, 24000)

The audio is then saved into the output_file as a .wav file, concluding the single responsibility of the Talker class: text-to-speech generation.
