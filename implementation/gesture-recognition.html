AI-Autostart & SoundLab Game - Hand Gesture Recognition

Google's library mediapipe is used for hand gesture recognition. This library is used in both AI-Autostart and SoundLab Game player. In both cases the corresponding video gesture recognition classes create mediapipe's GestureRecognizer with the following options, allowing for real-time input using the opencv library to capture webcam input. It is important to notice that method self._result_callback is injected as a parameter. This method is run asynchronously by the GestureRecogniser.

def _create_recognizer(self):
   options = GestureRecognizerOptions(
       base_options=BaseOptions(model_asset_path=self.model_path),
       running_mode=RunningMode.LIVE_STREAM,
       result_callback=self._result_callback,
   )
   return GestureRecognizer.create_from_options(options)

The implementation of the _result_callback differs in the two systems.

SoundLab Game player

In the SoundLab Game player, the public facing function is get_gesture. This is a blocking function, and its functionality has been inspired by the input function in python, except that it does not return the text written into the terminal, but the gesture shown by the user. The only argument of this function is gestures_to_spot. In most of the cases, the SoundLab Game player is looking for a subset of all gestures, and this is the list of the hand gestures it should be looking for.

def get_gesture(self, gestures_to_spot: list[EnumGesture]) -> EnumGesture:
   """
   Start the gesture recognition process and return the first gesture detected that is in the gestures_to_spot list.
   :param gestures_to_spot:
   :return:
   """
   self._reset(gestures_to_spot)
   self._start_recognition()
   return self._get_last_gesture()

Since the _result_callback method is run asynchronously, the state of VideoGestureRecogniser has the attribute _is_running. As soon as the _result_callback function recognises a gesture from the _gestures_to_spot list, it stops the main loop by setting the variable _is_running to False.

def _result_callback(self, result: GestureRecognizerResult, output_image: mp.Image, timestamp_ms: int):
   """
   Run for each picture analysed by the recognizer. If a gesture to spot is detected, stop the recognition.
   """
   if len(result.gestures) < 1:
       # then no hand detected
       return
   # set the last gesture
   self._last_gesture_category = result.gestures[0][0].category_name
   self._last_handedness = result.handedness[0][0].category_name


   # victory set as a special case for stopping the loop
   if self._get_last_gesture() in self._gestures_to_spot:
       self._stop()

AI-Autostart

In the AI-Autostart, the VideoGestureRecogniser class implements the Observer pattern. When a gesture is recognised, its subscribers, PowerManager and GestureController are updated.

def _result_callback(self, result: GestureRecognizerResult, output_image: mp.Image, timestamp_ms: int):
   """
   Run for each picture analysed by the recogniser.
   """
   if len(result.gestures) < 1:
       self.update_subscribers(None)
       return


   gesture_name = result.gestures[0][0].category_name
   handedness = ""
   if result.handedness and result.handedness[0]:
       handedness = result.handedness[0][0].category_name


   gesture = EnumGesture.from_gesture(gesture_name, handedness)
   if gesture == EnumGesture.INVALID:
       self.update_subscribers(None)
       return


   self.update_subscribers(gesture.value)

This makes the VideoGestureRecogniser a very portable class, that has no information about its subscribers, while allowing the gesture recognition and the control to be running in one thread if needed.

Because the gesture recognition on AI-Autostart is intended to run for an extended period of time continuously, low power mode has been implemented. For this purpose a utility class Fps has been created. The main used method is its is_time_for_next_frame:

def is_time_for_next_frame(self) -> bool:
   """
   Uses the time since last frame to determine if it's time for next frame.
   """
   if self.elapsed_since_last_frame() > 1./self._setFps:
       self._prev = time.time()
       return True
   return False

This method is used in VideoGestureRecogniser's _capture_frame_method, skipping frames if necessary:

def _capture_frame(self, cap):
   """
   Capture a frame from the camera, ensure it's valid and skip frames as necessary.
   """
   ret, frame = cap.read()
   if not ret or frame is None or frame.size == 0 or not self.fps_manager.is_time_for_next_frame():
       return None
   self.fps_manager.update()
   return frame

This returned None is then used in the recogniser's main loop to skip recognition altogether:

while cap.isOpened() and self.isRunning:
   frame = self._capture_frame(cap)
   if frame is None:
       # in low power mode
       continue


   cropped_frame = self._process_person_detection(frame)
   self._send_to_recogniser(cropped_frame, recognizer)
   self._display_frame(frame)

This FPS management therefore allows us to send frames to the recognizer only once per second, reducing the load of the hand gesture recognition on the hardware.

The actual control of the low power mode is performed by the PowerManager class, which is subscribed and continuously updated by the VideoGestureRecogniser itself.

The update method of the PowerManager keeps track of the Open_palm gesture, which is used to deactivate the low power mode:

def update(self, update: str | None):
   """
   Args:
       update: Detected gesture name, or None when no gesture is detected.
   """
   now = time()


   # no gesture detected
   if update is None:
       self.open_palm_start_time = None
       self.try_activate_LPM(now)
       return


   self.last_gesture_time = now


   # open palm must be held continuously to deactivate LPM
   if update in {EnumGesture.OPEN_PALM.value, "Open_Palm"}:
       if self.open_palm_start_time is None:
           self.open_palm_start_time = now


       self.try_deactivate_LPM(now)
       return


   # other gestures interrupt the hold timer
   self.open_palm_start_time = None

Methods try_activate_LPM and try_deactivate_LPM perform a check to see if enough time has passed to activate/deactivate the low power mode.
