Future Work

New Functionalities

1. Cross-platform support (both products)

Both AI-Autostart and No-UI Game are currently built and tested on Windows. A natural next step is to extend support to macOS.

2. Language support — No-GUI Game

The game engine currently generates and narrates games in British English only. Adding support for additional languages (Spanish, Chinese etc.) would significantly broaden the accessibility of the platform.

As explained in the Text-to-speech implementation section, hexgrad/kokoro is used for speech generation — specifically an object of class KPipeline. The constructor of this class takes in a language code for the language it should generate speech in. In future work, this could be expanded by giving the method _get_pipeline a parameter lang_code, that would allow the frontend to choose what language code to use it with.

def _get_pipeline(self) -> KPipeline:
   if self.pipeline is None:
       self.pipeline = KPipeline(lang_code='b', repo_id="hexgrad/Kokoro-82M")
   return self.pipeline

The names of the voices as well as the speakers would need to be changed correspondingly on the frontend side. See VOICES.md [1] from the official Kokoro team for more detail about support of different voices.

Regarding the frontend changes, currently in method title_row, there is a list VOICES containing all the names of the British English voices Kokoro offers. This list might become a dictionary, mapping language code to the list of voices. A dropdown similar to the one for choice of a voice would be implemented, allowing the user to choose a language — the voices dropdown would then show the voices available.

The last thing needed would be the preview audios. There is no standardised method, but the main sample code from Talker gives a good example of how to use and generate speech using the Talker class.

3. On-Screen Text Display — No-GUI Game

Currently the game player is entirely audio-based with no text shown on screen. A planned extension is to make the narration text optionally visible on screen, with a toggle in the settings page to switch it on or off. When enabled, the current scene text and options would be displayed in the player window as they are narrated, improving the experience of sighted users.

4. ReadingStar Integration

A further extension targeted at autistic children and partially sighted users would be integrating with ReadingStar, a karaoke-style reading tool designed for autistic children, to provide karaoke-style word highlighting. As the TTS narration progresses through the scene text, the word or phrase currently being spoken is highlighted in real time.

Existing Bugs

1. AI-Autostart — camera toggle on file action rows does not work

In the File Opening settings tab, each row has a Uses Camera toggle intended to keep camera handoff behaviour enabled after a file is launched, but it currently has no effect. As a result, AI-Autostart cannot be used to open applications that require the camera.

2. No-GUI Game Player — audio does not stop when GUI is turned off

When the GUI is manually turned off during playback, the audio does not stop.
