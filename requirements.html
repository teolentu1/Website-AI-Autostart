Project Specification & Requirements

Project Overview

Summary

AI-Autostart is a Windows-based system tray application designed to improve accessibility across digital interactive systems by enabling hands-free, adaptive interaction. The project focuses on users whose accessibility needs are not adequately addressed by current interfaces, including blind and visually impaired users, wheelchair users, neurodivergent individuals, and users with limited motor mobility.

The system operates as a lightweight background application that continuously monitors the environment using a combination of Computer Vision and Intel Wi-Fi Proximity Sensing. Based on detected user presence and interaction intent, AI-Autostart automatically initiates interaction without requiring physical contact.

To demonstrate the applicability of the platform, AI-Autostart connects to two prototypes:

  • An adaptive self-service kiosk UI prototype
  • A fully playable no-UI game for blind and visually impaired users
Project Objectives
  • Design and implement a Windows system tray accessibility platform
  • Enable reliable touch-free interaction initiation
  • Develop an adaptive self-service kiosk UI prototype
  • Implement a fully playable no-UI game for blind users
  • Integrate a fully local Large Language Model for narration
  • Deliver a single standalone Windows executable
Project Scope

In Scope

AI-Autostart focuses on the design and implementation of a general accessibility platform using Computer Vision and Intel Wi-Fi Proximity Sensing™. The project includes a system tray application, an adaptive kiosk UI prototype, and a no-UI game for blind and visually impaired users.

Out of Scope

Mobile platforms, macOS, and Linux are out of scope. The system does not store or transmit user data, and large-scale commercial deployment is not addressed.

Background and Motivation

Context of the Problem

Despite increasing awareness of digital accessibility, many interactive systems deployed in public environments fail to adequately support users with diverse accessibility needs. Accessibility is often implemented as an optional feature rather than a core design principle.

Motivation

AI-Autostart aims to move beyond add-on accessibility solutions by enabling proactive, touch-free interaction initiation. The system detects user presence and intent automatically, reducing cognitive load and increasing independence.
